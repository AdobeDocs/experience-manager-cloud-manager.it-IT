# Manage your Dispatcher Configurations {#manage-your-dispatcher-configurations}

## Using Cloud Manager to Deploy Dispatcher Configuration Files {#using-cloud-manager-to-deploy-dispatcher-configuration-files}

Cloud Manager is able to deploy web server and Dispatcher configuration files assuming they are stored in the Git Repository, in addition to normal AEM content packages.****

To take advantage of this capability, the Maven build should produce a zip file which contains at least two directories - ***conf*** and ***conf.d***. This zip file can be produced using the maven-assembly-plugin. Projects generated by Cloud Manager using the built-in wizard have the correct Maven project structure created as part of the project creation. [](create-an-application-project.md) Questo è il percorso consigliato per i nuovi clienti dei servizi gestiti.

Dopo la distribuzione a un’ **istanza** dispatcher, il contenuto di queste directory sovrascriverà il contenuto di tali directory nell’istanza Dispatcher. Poiché i file di configurazione del server Web e del dispatcher spesso richiedono informazioni specifiche per l’ambiente, per poter utilizzare correttamente questa funzionalità, è necessario innanzitutto lavorare con i tecnici del successo cliente (CSE) per impostare queste variabili di ambiente in ***/etc/sysconfig/httpd***.

### Passaggi per la configurazione del dispatcher per i clienti dei servizi gestiti esistenti {#steps-for-configuring-dispatcher}

Per completare il processo iniziale nella configurazione del dispatcher, effettuate le operazioni seguenti:

1. Ottenete i file di configurazione di produzione correnti dal vostro CSE.
1. Rimuovete i dati specifici dell'ambiente codificato (ad esempio, IP del renderer di pubblicazione) e sostituiteli con le variabili.
1. Definite le variabili richieste nelle coppie chiave-valore per ciascun dispatcher di destinazione e richiedete al CSE di aggiungere a ***/etc/sysconfig/httpd*** in ogni istanza.
1. Verificate le configurazioni aggiornate nell’ambiente di visualizzazione, quindi richiedete al CSE di distribuirle in produzione.
1. Eseguire il commit dei file nell'archivio **Git**.

1. Distribuisci tramite Cloud Manager.

>[!NOTE]
>
>Migrating Dispatcher and web server configurations to Git Repository may be done during Cloud Manager on-boarding, but can also be done at a later point in time.****

### Esempio {#example}

The specific file and directory structure may vary based on the specifics of your project, but this example should provide a concrete guide to how to structure your project to include Apache and Dispatcher Configurations.

1. Create a subdirectory named .`dispatcher`

   >[!NOTE]
   Potete utilizzare un nome qualsiasi, ma il nome della directory creato in questo passaggio deve essere uguale al nome utilizzato al punto 6.

1. Questa sottodirectory conterrà un modulo Maven che crea il file zip del Dispatcher utilizzando il plug-in di assemblaggio Maven. To start this, in the  directory, create a  file with this content, changing the parent reference, artifactId, and name as necessary.`dispatcher``pom.xml`

   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <project xmlns="http://maven.apache.org/POM/4.0.0"
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
           xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/maven-v4_0_0.xsd">
       <modelVersion>4.0.0</modelVersion>
       <parent>
           <!-- reference to your parent pom -->
       </parent>
   
       <artifactId>dispatcher</artifactId> <!-- feel free to change this -->
       <packaging>pom</packaging>
       <name>dispatcher</name> <!-- feel free to change this -->
   
       <build>
           <plugins>
               <plugin>
                   <groupId>org.apache.maven.plugins</groupId>
                   <artifactId>maven-assembly-plugin</artifactId>
                   <version>3.1.0</version>
                   <executions>
                       <execution>
                           <phase>package</phase>
                           <goals><goal>single</goal></goals>
                           <configuration>
                               <descriptors>
                                   <descriptor>assembly.xml</descriptor>
                               </descriptors>
                               <appendAssemblyId>false</appendAssemblyId>
                           </configuration>
                       </execution>
                   </executions>
               </plugin>
           </plugins>
       </build>
   </project>
   ```

   >[!NOTE]
   Come nel passaggio 1, artifactId e name qui possono essere altri valori se si desidera; `dispatcher` ecco un esempio usato per la semplicità.

1. Il plug-in di assemblaggio del cielo richiede un *descrittore* per definire la modalità di creazione del file zip. Per creare questo descrittore, create un file (ancora una volta, nella `dispatcher` sottodirectory) denominato `assembly.xml`con questo contenuto. Si noti che a questo nome di file viene fatto riferimento alla riga 26 nel `pom.xml` file precedente.

   ```xml
   <assembly xmlns="http://maven.apache.org/ASSEMBLY/2.0.0"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://maven.apache.org/ASSEMBLY/2.0.0 http://maven.apache.org/xsd/assembly-2.0.0.xsd">
     <id>distribution</id>
     <formats>
       <format>zip</format>
     </formats>
     <includeBaseDirectory>false</includeBaseDirectory>
     <fileSets>
       <fileSet>
         <directory>${basedir}/src</directory>
         <includes>
           <include>**/*</include>
         </includes>
         <outputDirectory></outputDirectory>
       </fileSet>
     </fileSets>
   </assembly>
   ```

1. A questo punto, create una sottodirectory denominata `src` (come riportato nel descrittore dell'assembly sopra alla riga 11) all'interno della sottodirectory dispatcher per memorizzare le configurazioni Apache e Dispatcher effettive. All'interno di questa `src` directory, create directory denominate `conf`, `conf.d`, `conf.dispatcher.d`e `conf.modules.d`.
1. Ora è possibile compilare le `conf`, `conf.d`, `conf.dispatcher.d`e `conf.modules.d` le directory con i file di configurazione. Ad esempio, la configurazione predefinita è costituita da questi file e da collegamenti simbolici.

1. Infine, nel file pom.xml nella directory principale del progetto, aggiungete un `<module>` elemento per includere il modulo dispatcher.

   Ad esempio, se l'elenco di moduli esistente è

   ```xml
       <modules>
           <module>core</module>
           <module>ui.apps</module>
           <module>ui.content</module>
       </modules>
   ```

   Dovrebbe modificarla in

   ```xml
       <modules>
           <module>core</module>
           <module>ui.apps</module>
           <module>ui.content</module>
           <module>dispatcher</module>
       </modules>
   ```

   >[!NOTE]
   Come indicato al punto 1, il valore dell' `<module>` elemento **deve** corrispondere al nome della directory creato.

1. Infine, per testare, eseguite il pacchetto mvn clean nella directory principale del progetto. Nell'output dovrebbero essere visualizzate linee come questa

   ```

   [INFO] Building zip: /Users/me/mycompany/dispatcher/target/dispatcher-1.0-SNAPSHOT.zip
   ```
